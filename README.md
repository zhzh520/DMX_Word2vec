# Word2vce

基于 **PyTorch** 从零实现 Word2Vec（Skip‑Gram + 负采样），以《红楼梦》和《聊斋志异》文言小说为语料，基于《红楼梦》的模型训练下对《聊斋志异》完整完成**数据预处理 → 模型训练 → 参数调优 → 多维度评估 → 最优模型应用**全流程，探索古典文言语料下词向量的语义表达能力与训练规律。

## 项目介绍

本项目从零实现 Word2Vec，针对**中国古典小说**做专项适配：

- 支持《聊斋志异》（文言短篇）、《红楼梦》（白话章回）两种典型文本
- 内置专用分词、停用词过滤、低频词筛选、负采样加速
- 提供多组参数对比实验与完整评估指标
- 输出可直接使用的 `.pt` 词向量模型

可用于：**古汉语语义挖掘、人物关系分析、文本分类、情节检索、文学研究**等。

## 功能特性

完整 pipeline：文本清洗 → 分词 → 建表 → 训练 → 评估 → 保存

支持 GPU 混合精度训练，速度大幅提升

聊斋 / 红楼梦专属词典，分词更准确

内置语义相似度、类比推理、聚类指标等多维度评估

自动绘制损失曲线、输出 JSON 实验报告

无向量坍缩、无过拟合的最优配置直接可用

##### 文本预处理差异

##### 预处理流程

1. 清洗：去数字、符号、空格、乱码
2. 分词：jieba + 古典小说专用词典
3. 过滤：移除停用词、低频词（min_count=8）
4. 构建：上下文窗口 + 负采样预生成

文言文专用预处理

```
import re
import jieba
from collections import Counter

def advanced_text_cleaning(text, is_classical=True):
    """古典文本专用清洗：区分文言/白话，保留特色词汇"""
    # 基础清洗：移除HTML/URL/乱码
    text = re.sub(r'<.*?>|http[s]?://\S+', '', text)
    
    # 保留中文+专属标点（文言保留更多特殊符号）
    if is_classical:
        text = re.sub(r'[^\u4e00-\u9fa5。，！？；：""''()（）【】《》\s]', '', text)
    else:
        text = re.sub(r'[^\u4e00-\u9fa5。，！？；：、\s]', '', text)
    
    # 移除多余空格 + 停用词过滤
    text = re.sub(r'\s+', ' ', text).strip()
    stop_words = {'的', '了', '在', '是', '之', '乎', '者', '也', '焉', '哉'} if is_classical else {'的', '了', '在', '是'}
    words = [word for word in text.split() if word not in stop_words]
    
    return ' '.join(words)

def build_vocab(tokenized_corpus, min_count=8):
    """构建词汇表：过滤低频词，适配古典文本"""
    word_counts = Counter([word for sentence in tokenized_corpus for word in sentence])
    vocab = {word: count for word, count in word_counts.items() if count >= min_count}
    
    # 索引映射（含PAD/UNK）
    idx_to_word = ['<PAD>', '<UNK>'] + list(vocab.keys())
    word_to_idx = {word: idx for idx, word in enumerate(idx_to_word)}
    
    print(f"词汇表大小: {len(word_to_idx)} | 过滤低频词数: {len(word_counts)-len(vocab)}")
    return word_to_idx, idx_to_word, vocab
```

## 思考题

1. **在训练Word2Vec模型时，词向量维度vector_size该如何设置？词向量维度数的设置和什么因素有关?**

   - **语料库规模**

     - 小语料（如仅《聊斋志异》全文，约 70 万字）：维度建议设置在**50-100**。语料量少，高维度向量无法被充分训练，易出现 “维度灾难”，50-100 维既能覆盖核心语义（如人物、鬼怪、情节相关词汇），又避免过拟合；
     - 大语料（如结合全古代小说语料，千万级以上）：可提升至 100-300 维，更高维度能编码更细粒度的语义（如区分 “报恩狐仙” 和 “作恶狐仙”）。

     **词汇表大小**

     词汇表越大（如《聊斋》中含生僻古汉语词汇，词汇表规模约 1 万），需要更高维度来区分不同词汇；若词汇表仅核心常用词（如 2000 个），50 维即可满足需求。

     **下游任务需求**

     - 简单任务（如词汇相似度计算、关键词匹配）：低维度（50-100）足够；
     - 复杂任务（如文本分类、情节情感分析）：需更高维度（100-200）以捕捉更多语义特征。

     **计算资源与效率**

     维度翻倍会导致：① 模型参数数量翻倍（Word2Vec 的参数主要为输入 / 输出权重矩阵，维度为 vector_size× 词汇表大小）；② 训练时间和内存占用近似翻倍。若仅用普通 CPU 训练《聊斋》模型，建议维度不超过 100，平衡效果与效率。

   词向量维度决定语义表达能力与计算量。维度太小会丢失语义，太大易过拟合、训练变慢。

   主要和语料大小、词汇量、下游任务、硬件资源有关：小语料（如《聊斋》）用 50–100 维，大语料可用 100–300 维。简单任务用低维，复杂任务适当提高，优先在效果和速度间做折中。
